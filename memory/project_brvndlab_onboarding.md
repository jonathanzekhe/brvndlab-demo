Formulaire d'onboarding à la première connexion du client. Basé sur le modèle d'un concurrent (screenshots à venir).

**Objectifs du formulaire :**
1. Connaître la situation actuelle du client (problématiques, niveau, niche)
2. Savoir où ils ont découvert Jonathan (quelle plateforme, quelle vidéo, quel contenu les a trigger)
3. Comprendre pourquoi ils sont venus à lui spécifiquement
4. Collecter des données exploitables pour la stratégie de contenu de Jonathan

**Email de contact affiché :** hey@brvndlab.com

**Où les données vont :**
- Toutes les réponses arrivent dans le dashboard coach de Jonathan
- Chaque client a ses réponses d'onboarding visibles dans sa fiche
- Plus tard : dashboard agrégé pour voir les tendances (d'où viennent les clients, quels contenus convertissent)

**Flow :**
- Le client se connecte pour la première fois
- Avant d'accéder au Brand OS, il doit remplir le formulaire d'onboarding
- Une fois rempli, il accède à l'app normalement
- Le formulaire n'apparaît plus jamais

**Why:** Jonathan veut savoir ce qui amène ses clients à lui pour optimiser sa création de contenu et son acquisition.

**How to apply:** Créer une page /onboarding qui s'affiche avant le dashboard si le client n'a pas encore rempli le formulaire. Les données sont stockées dans Convex et visibles dans le Coach Dashboard.
